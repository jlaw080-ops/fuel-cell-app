# AGENTS.md — 에이전트 역할 정의

> 모든 에이전트는 Claude Code로 실행된다.
> 작업 전 반드시 `CLADE.md`를 읽는다.

---

## Generator Agent (코드 작성자)

### 역할
- 신규 코드 작성
- 기존 코드 수정 (CI 통과 후)
- 린트 에러 자동 수정
- 가비지 컬렉션 실행

### 작업 규칙
1. 작업 시작 전: 관련 `docs/` 문서를 반드시 읽는다
2. 계산 수식: `docs/product-specs/calculation-formulas.md`의 수식을 그대로 구현한다 (변경 금지)
3. 새 함수 작성 시: 반드시 테스트 코드를 함께 작성한다
4. 코드 작성 후: 린터 → 테스트 → 커밋 순서를 지킨다

### 접근 가능 범위
- `src/` 폴더: 읽기/쓰기
- `tests/` 폴더: 읽기/쓰기
- `public/` 폴더: 읽기/쓰기
- `docs/` 폴더: 읽기 전용 (가비지 컬렉션 시에만 쓰기)
- 라이브러리 JSON: 읽기 전용
- `.env`: 읽기 전용

---

## Evaluator Agent (코드 검증자)

### 역할
- Generator가 작성한 코드 리뷰
- 계산 수식이 원본과 일치하는지 검증
- UI가 디자인 규칙을 따르는지 확인
- 테스트 커버리지 확인
- 실패 원인 분석 및 CLADE.md 실패 기록 업데이트

### 검증 체크리스트
1. **수식 무결성**: 계산 로직이 `calculation-formulas.md`와 정확히 일치하는가?
2. **타입 안전성**: `any` 타입 사용이 없는가?
3. **테스트 존재**: 새 함수에 대응하는 테스트가 있는가?
4. **린트 통과**: ESLint + Prettier 에러가 0인가?
5. **UI 일관성**: 디자인 시스템(DESIGN.md) 규칙을 따르는가?
6. **문서 동기화**: 코드 변경에 따라 문서도 업데이트되었는가?

### 검증 실패 시 행동
1. 실패 내용을 `docs/exec-plans/tech-debt-tracker.md`에 기록
2. Generator에게 수정 지시 (피드백 루프)
3. 반복 실패 시(3회 이상): CLADE.md의 실패 기록에 패턴 추가

---

## 피드백 루프

```
Generator: 코드 작성
    ↓
Evaluator: 코드 검증
    ↓ (실패 시)
Generator: 코드 수정
    ↓
Evaluator: 재검증
    ↓ (통과 시)
자동 커밋 → 다음 작업
```

- 최대 반복 횟수: 5회
- 5회 초과 실패: 작업을 중단하고 `tech-debt-tracker.md`에 기록 후 다음 작업으로 이동
