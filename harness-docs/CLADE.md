## 🚫 절대 해서는 안 되는 것 (NEVER)

### 최우선 금지
- **계산 수식 임의 변경 금지**: `docs/product-specs/calculation-formulas.md`에 정의된 수식은 절대 수정하지 않는다
- **라이브러리 JSON 구조 임의 변경 금지**: JSON 파일의 키 이름, 중첩 구조를 변경하지 않는다

### 파일 관련
- CI 테스트 미통과 상태에서 기존 파일 수정 금지
- 파일 삭제 금지 (deprecated 처리만 허용)
- `.env` 파일에 실제 키 값 하드코딩 금지

### 코드 관련
- `any` 타입 사용 금지 (TypeScript)
- 테스트 없이 신규 함수 작성 금지
- 콘솔 로그를 프로덕션 코드에 남기지 않는다
- 외부 패키지를 임의로 추가하지 않는다 (docs/references/ 에 등록된 것만 사용)

### UI 관련
- 디자인 시스템(docs/DESIGN.md)에 정의된 색상/폰트/간격을 임의로 변경하지 않는다
- 탭 구조(5탭)를 변경하지 않는다

## 🔄 작업 흐름 (WORKFLOW)

```
코드 작성 → 린터 검사 → 에러 시 직접 수정 → 린터 재검사
→ 통과 시 테스트 실행 → 통과 시 커밋 → 다음 작업 진행
```

1. 코드 작성 전: 관련 docs/ 파일을 읽는다
2. 코드 작성 후: `npm run lint` 실행
3. 린트 에러 시: 직접 수정 후 재실행 (최대 3회 반복)
4. 린트 통과 후: `npm run test` 실행
5. 테스트 통과 후: 커밋 (프리커밋 훅이 자동 재검사)
6. 테스트 실패 시: 에러 원인을 `docs/exec-plans/tech-debt-tracker.md`에 기록
