# 설계 문서 (Design Docs)

## 핵심 설계 원칙 (`core-beliefs.md` 참조)

1. 수식 무결성이 최우선이다
2. 단순하고 직관적인 UI
3. 데이터는 항상 DB에 저장/복원 가능
4. 모든 계산은 서버 사이드에서 수행

## 설계 결정 기록

| 결정 | 선택 | 이유 |
|------|------|------|
| 페이지 구조 | 단일 페이지 + 5탭 | 1인 사용, 탭 간 데이터 연결성 |
| DB | Supabase | 무료, Vercel 연동 용이 |
| AI | Gemini API | 사용자 지정 |
| PDF | 서버사이드 생성 | 일관된 출력 품질 |
