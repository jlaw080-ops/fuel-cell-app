# 핵심 설계 신념 (Core Beliefs)

## 1. 수식은 성역이다
- 계산 수식은 `calculation-formulas.md`에서만 정의한다
- 코드는 수식을 "구현"할 뿐, "해석"하거나 "개선"하지 않는다
- 수식 변경이 필요하면 사람이 직접 문서를 수정한 후 코드를 업데이트한다

## 2. 데이터는 항상 살아있다
- 사용자 입력, 계산 결과는 모두 Supabase에 저장
- 브라우저를 닫아도 마지막 상태를 복원할 수 있어야 한다
- 프로젝트를 여러 개 만들어서 비교할 수 있어야 한다

## 3. 에러는 자산이다
- 실패한 코드 패턴은 CLADE.md에 기록한다
- 에러 원인은 린트 규칙으로 변환한다
- 같은 실수를 두 번 하지 않는 구조를 만든다

## 4. 문서와 코드는 하나다
- 문서 없는 코드는 작성하지 않는다
- 코드 변경 시 관련 문서를 반드시 업데이트한다
- 가비지 컬렉션 시 문서-코드 불일치를 점검한다
